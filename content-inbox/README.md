# Content Inbox

Put future website content here first. I can then map each item into the al-folio files for you.

## Folder Map

- `about/`: short bio, profile photo notes, contact text, current focus.
- `projects/`: project descriptions, screenshots, demo links, GitHub links.
- `cv/`: education, experience, skills, awards, certificates, resume PDF.
- `blog/`: drafts or notes that should become blog posts.
- `publications/`: BibTeX, posters, reports, preprints, paper PDFs.
- `assets/`: photos, screenshots, PDFs, icons, and other files.

## Site Mapping

- About page -> `_pages/about.md`
- Project cards and pages -> `_projects/*.md`
- CV page -> `_data/cv.yml` and `_pages/cv.md`
- Blog posts -> `_posts/YYYY-MM-DD-title.md`
- Publications -> `_bibliography/papers.bib` and `_pages/publications.md`
- Images -> `assets/img/`
- PDFs -> `assets/pdf/`

## Recommended Workflow

1. Drop raw content into the matching folder.
2. Keep filenames descriptive, for example `ai-chatbot-project.md` or `resume-2026.pdf`.
3. If an image belongs to a project, put the project name in the image filename.
4. Ask me to import the content, and I will convert it into the correct al-folio format.
